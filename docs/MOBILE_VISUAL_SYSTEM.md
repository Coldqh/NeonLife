# Mobile Visual System

## Режим отображения

Новая оболочка активируется при ширине до 860 px. Десктопная оболочка остаётся в проекте и продолжает использовать прежние рабочие пространства.

## Источники данных

Интерфейс не содержит отдельного демонстрационного мира:

- профиль читает `GameSession.player`, `people`, `pressure`, `vehicles`;
- глобальная карта читает весь `metropolitan.sectors`;
- границы районов строятся из `sector.districtId`;
- магистрали читаются из `roadNodes` и `roadLinks`;
- станции читаются из `transitStations`;
- локальная карта читает `urban.buildings`, `metropolitan.locations` и `localScene.playerPosition`;
- Nearby читает `localScene.actors`, `localScene.buildings`, `vehicles.vehicles` и локальные события;
- перемещение использует существующий `getTravelOptions` и `travelToLocation`.

## Масштаб

Число секторов не зашито в интерфейс. Карта использует `metropolitan.config.sectorsWide`, `sectorsHigh` и полный массив `metropolitan.sectors`. При текущей конфигурации отображаются 1 512 секторов.

## Локальная карта

Каждый сектор имеет собственный детерминированный seed. По нему строится стабильная сетка улиц. Для активного сектора поверх неё выводятся реальные материализованные здания, локации и позиция игрока. Для холодного сектора используется стабильная квартальная раскладка до его материализации.

## Ограничение изменений

Патч не добавляет новых игровых действий и не меняет расчёты экосистемы. Он только предоставляет новый мобильный способ читать уже существующее состояние и запускать уже существующие действия.

## 0.30.1 — интерактивность

- видимая кнопка всегда имеет обработчик;
- невозможное действие скрывается либо заменяется текстом с причиной;
- портрет персонажа определяется функцией от постоянного ID;
- выбранный контакт раскрывается внутри мобильного экрана;
- интерфейс не использует `text-overflow: ellipsis` для игровых данных;
- мобильный контейнер использует полную ширину устройства до 540 px.
