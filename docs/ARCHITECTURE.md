# NEON LIFE — архитектура v0.1.0

## Граница первого среза

Версия `0.1.0` строит рабочую оболочку **SEVEN DAYS BELOW**. Она не симулирует весь город, но уже задаёт границы, через которые дальнейшие системы будут подключаться без переписывания интерфейса.

## Слои

### `app/`

Композиция приложения и layout. Здесь нет генерации мира, экономики или логики NPC.

### `core/`

Независимый фундамент:

- игровое время;
- события;
- стабильные ID;
- seeded random;
- хранение локальных настроек;
- базовые типы симуляции.

Код из `core/` не зависит от React-компонентов.

### `world/`

Метаданные и будущая генерация города. В текущем срезе создано детерминированное состояние демонстрационного мира.

### `people/`

Модели и демонстрационные данные людей. Скрытые параметры NPC не выводятся напрямую в UI — интерфейс показывает известные герою факты.

### `gameplay/`

Состояние героя и единый контракт `WorldAction`. Работы, поездки, разговоры, лечение и будущий бой должны использовать этот контракт.

### `ui/`

Дизайн-система, компоненты, темы и адаптивность. UI получает состояние и команды извне, а не хранит правила мира внутри карточек.

## Состояние в v0.1.0

- UI-настройки сохраняются в `localStorage`.
- Демонстрационная сессия также сохраняется локально для проверки непрерывности интерфейса.
- Полноценные сохранения мира должны перейти в IndexedDB на следующем техническом этапе.

## Детерминизм

`SeededRandom` является единственной допустимой основой случайности для генерации мира. Стабильные сущности получают ID через `createStableEntityId(namespace, seed)`.

## Следующий технический шаг

1. IndexedDB и миграции сохранений.
2. Command bus для действий игрока.
3. Очередь будущих событий.
4. Реальный `WorldState` вместо демонстрационного session state.
5. Один район с локациями, расписаниями и перемещением.
