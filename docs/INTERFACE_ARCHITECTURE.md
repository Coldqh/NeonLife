# Unified interface architecture

## Один интерфейс

`App.tsx` монтирует только `GameShell`. Разделения на отдельные desktop и mobile React-деревья больше нет. Адаптация выполняется CSS внутри одной структуры.

## Навигация

Единственный источник состояния экрана — `GameScreen`:

- `profile`;
- `map`;
- `nearby`.

Карта является стартовым экраном и центральной кнопкой навигации. «Главная» и «Путь» не существуют ни визуально, ни в типах.

## Транспорт

Когда `GameSession.transit.player.journey` существует, `TransitJourneyScreen` монтируется поверх всего приложения. Он скрывает обычную навигацию функционально, а не CSS-подменой. Все действия вызывают существующие команды Transit Operations.

## Карты

`GlobalCityMap` читает `MetropolitanState` и отвечает только за viewport, выбор сектора и слои.

`LocalSectorMap` читает:

- `metropolitan.roadLinks` и `roadNodes`;
- `urban.buildings`;
- `metropolitan.locations`;
- `localScene.playerPosition`.

Компонент не создаёт fallback-здания и вымышленные улицы. Полноценная постоянная уличная топология остаётся отдельным следующим доменным патчем.

## Ограничения файлов

- `App.tsx` — не более 220 строк;
- любой UI TS/TSX/CSS-файл — не более 600 строк;
- новые экраны и стили разделяются по функциональной ответственности.
